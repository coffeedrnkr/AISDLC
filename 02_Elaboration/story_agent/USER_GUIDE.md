# Story Writing Agent - User Guide

This guide describes how to use the **Story Writing Agent** to decompose Epics into high-quality, "INVEST"-compliant User Stories with Gherkin Acceptance Criteria.

## Methodology

### 1. The 3 C's (Card, Conversation, Confirmation)
*   **Card**: The physical token (or Jira issue) holding the requirement.
*   **Conversation**: The collaboration to refine details (this Agent simulates the first round of this).
*   **Confirmation**: The Acceptance Criteria (Gherkin) that confirm the story is done.

### 2. INVEST Criteria
We ensure every generated story is:
*   **I**ndependent
*   **N**egotiable
*   **V**aluable
*   **E**stimable
*   **S**mall
*   **T**estable

### 3. Gherkin (BDD)
We use the **Given-When-Then** format for "Test-First" development:
*   **Given** [Initial Context]
*   **When** [Action]
*   **Then** [Outcome]

## Inputs

1.  **Epic**: The parent Epic markdown file (generated by the Epic Agent).
2.  **Architecture**: Technical constraints (from Architecture Agent).

## Outputs

*   **Markdown Files**: `outputs/stories/Story-[ID]-[Title].md`
*   **Jira Issues**: If `--sync` is enabled, stories are pushed to Jira.

## How to Run

### Option A: Terminal (Standard)
```bash
python story_agent/story_agent.py --epic agents/epic_agent/outputs/epics/epic_USR_001.md --output agents/story_agent/outputs
```

### Option B: Gemini Slash Command
```
/story-gen
```

### Option C: Natural Language (MCP)
> "Generate user stories for this Epic."
> "Turn this elaboration into Gherkin scenarios."

## Review Workflow
1.  **Generate**: Run the agent.
2.  **Review**: Open the generated Markdown files.
    *   *Check*: Are the scenarios realistic?
    *   *Refine*: Edit the Gherkin scenarios if needed.
3.  **Sync**: Run with `--sync` again to push updates to Jira.
